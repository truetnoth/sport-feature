# Навигатор упражнений — Спортивная редакция Т—Ж

## Суть проекта

Интерактивный навигатор по 600+ упражнениям спортивной редакции Т—Ж. Читатель через систему фильтров подбирает упражнения по целям, оборудованию, группам мышц и особенностям здоровья. Может сохранять упражнения в избранное и формировать плейлисты тренировок.

## Референс дизайна

**Главный референс**: https://t-j.ru/entry-rules-travel-navigator/

Дизайн навигатора упражнений должен максимально повторять стиль и UX этого навигатора стран:
- Фильтры сверху (чипы, тогглы, дропдауны)
- Сетка карточек с фото и метаданными
- Чистый белый фон, жёлтый акцент (бренд Т—Ж)
- Группировка результатов по категориям
- Шапку сайта и левый сайдбар можно показать серыми плейсхолдерами или не показывать

## Архитектура данных

### Модель упражнения (Exercise)
```json
{
  "id": "ex_001",
  "title": "Жим штанги лёжа",
  "slug": "zhim-shtangi-lyozha",
  "shortDescription": "Базовое упражнение для развития грудных мышц и трицепсов",
  "fullDescription": "Подробное описание техники выполнения...",
  "image": "/images/ex_001.jpg",
  "gif": "/images/ex_001.gif",
  "sourceArticleUrl": "https://t-j.ru/...",
  "sourceArticleTitle": "Название статьи-источника",
  "tags": {
    "goal": ["strength"],
    "equipment": ["barbell", "bench"],
    "bodyPart": ["chest", "triceps", "shoulders"],
    "location": ["gym"],
    "difficulty": "intermediate",
    "special": []
  }
}
```

### Система тегов (фильтров)

#### 1. Цель тренировки (`goal`)
- `strength` — Для развития силы и роста мышц
- `flexibility` — Для развития гибкости и подвижности суставов
- `coordination` — Для развития координации
- `endurance` — Для развития выносливости и тренировки сердечно-сосудистой системы
- `health` — Для поддержания здоровья при заболеваниях и особенных состояниях
- `yoga` — Из йоги

#### 2. Оборудование (`equipment`)
- `none` — Без оборудования
- `dumbbells` — С гантелями
- `barbell` — Со штангой
- `kettlebell` — С гирями
- `machines` — В тренажёрах
- `fitball` — С фитболом
- `chair` — Со стулом
- `pullup_bar` — На турниках
- `trx` — С петлями TRX
- `resistance_bands` — С фитнес-резинками

#### 3. Группа мышц (`bodyPart`) — актуально для goal=strength + equipment=machines
- `chest` — На грудь
- `shoulders` — На плечи
- `biceps` — На бицепсы
- `triceps` — На трицепсы
- `back` — На спину
- `abs` — На пресс
- `glutes` — На ягодицы
- `legs` — На ноги

#### 4. Область тела (`bodyArea`) — актуально для goal=flexibility
- `spine` — Для спины
- `lower_body` — Для нижней части тела
- `upper_body` — Для верхней части тела

#### 5. Особые состояния (`special`) — актуально для goal=health
- `pregnant` — Для беременных
- `elderly` — Для пожилых
- `herniated_disc` — При межпозвонковой грыже
- `knee_problems` — При заболеваниях коленного сустава
- `hip_problems` — При заболеваниях тазобедренного сустава
- `posture` — Для красивой осанки
- `foot_health` — Для здоровья стоп
- `office_warmup` — Для разминки в офисе

#### 6. Место выполнения (`location`)
- `home` — Дома
- `gym` — В зале
- `outdoor` — На улице
- `office` — В офисе

#### 7. Сложность (`difficulty`)
- `beginner` — Начинающий
- `intermediate` — Средний
- `advanced` — Продвинутый

## Функциональность MVP

### Обязательно (MVP)
1. **Фильтрация** — мульти-выбор по всем категориям фильтров. Все фильтры видны сразу (без зависимой логики показа/скрытия в v1)
2. **Карточки упражнений** — фото/заглушка, название, краткое описание, теги-бейджи
3. **Детальная страница упражнения** — отдельный роут с полным описанием, пошаговой инструкцией, гифкой, тегами, ссылкой на статью-источник, кнопками избранного/плейлиста, блоком похожих упражнений (референс: Jefit)
4. **Избранное** — сохранение упражнений в избранное (localStorage)
5. **Плейлисты** — создание именованных списков упражнений, добавление/удаление из них
6. **Счётчик результатов** — «Нашли N упражнений»
7. **Поиск** — текстовый поиск по названию и описанию

### Желательно (v2)
- Экспорт плейлиста (PDF / ссылка)
- Таймер для выполнения упражнений
- Просмотренные / выполненные упражнения
- Рандомная тренировка по фильтрам

## Технический стек

- **React** (Vite или CRA)
- **TypeScript** предпочтительно
- **CSS Modules** или **Tailwind CSS**
- **localStorage** для хранения избранного и плейлистов
- **JSON-файл** как источник данных (легко подменить на API)
- Деплой: **GitHub Pages**

## Подключение реальных данных

Когда будет готова Google-таблица с упражнениями:
1. Экспортировать таблицу в CSV
2. Конвертировать CSV → JSON по схеме Exercise выше
3. Заменить `exercises.json` на реальные данные
4. Изображения загрузить в `/public/images/` или использовать CDN

Скрипт конвертации из CSV в JSON нужно будет написать отдельно.

## Принятые решения

1. **Данные** — Google-таблицы пока нет. Используем рыбный текст (lorem ipsum) и плейсхолдеры вместо изображений. JSON-файл с моковыми данными как источник, который потом заменим на реальный.
2. **Изображения** — плейсхолдеры (цветные прямоугольники с названием упражнения или SVG-иконки). Когда появятся реальные фото — подставим.
3. **Фильтры** — все фильтры видны сразу, без зависимой логики показа/скрытия. Упрощает MVP, зависимость фильтров можно добавить в v2.
4. **Йога** — информация будет предоставлена позже. Пока йога — просто один из тегов `goal`, без отдельной подсистемы.
5. **Источники** — одна статья может содержать несколько упражнений. Ссылка на статью-источник указывается внутри детальной карточки упражнения.

## Страница детального просмотра упражнения

**Референс**: https://www.jefit.com/exercises/86/cable-lat-pulldown-wide-grip

При клике на карточку упражнения открывается отдельная страница (роут `/exercise/:slug`), а НЕ модалка. Структура страницы сверху вниз:

### Шапка упражнения
- Кнопка «Назад» (возврат к списку с сохранением фильтров)
- Название упражнения (h1)
- Теги-бейджи: цель, оборудование, группы мышц, сложность

### Медиа-блок
- Основное изображение / GIF-анимация выполнения (крупно, на всю ширину контентной области)
- Под ним — миниатюры дополнительных фото (галерея, 2–3 ракурса)
- В MVP: цветные плейсхолдеры с подписью «GIF» / «Фото 1» / «Фото 2»

### Блок инструкции
- Заголовок «Как выполнять»
- Пошаговая инструкция (нумерованный список)
- В MVP: рыбный текст с 3–4 шагами

### Блок метаданных
- **Группы мышц**: основная + вспомогательные (с иконками-силуэтами если возможно)
- **Оборудование**: что нужно
- **Сложность**: beginner / intermediate / advanced
- **Тип**: силовое / растяжка / кардио и т.д.

### Блок действий
- Кнопка «❤️ В избранное» (toggle)
- Кнопка «＋ В плейлист» → дропдаун с выбором существующего плейлиста или создание нового
- Ссылка «Читать полную статью на Т—Ж» → внешняя ссылка на sourceArticleUrl

### Блок «Похожие упражнения»
- Горизонтальная карусель или сетка 3–4 карточек
- Подбор по совпадению тегов (та же группа мышц или то же оборудование)

## Навигация приложения (роутинг)

- `/` — главная страница с фильтрами и сеткой карточек
- `/exercise/:slug` — детальная страница упражнения
- `/favorites` — страница избранного (опционально, можно панелью)
- `/playlist/:id` — просмотр конкретного плейлиста

Использовать `HashRouter` для совместимости с GitHub Pages.
